# go.sh & go.env 사용 설명서 (형들을 위한 뽀송 버전) 👨‍💻👩‍💻

형/누나들 안녕! 👋 리눅스 쓰다 보면 맨날 치는 명령어 또 치고, 설정 파일 찾느라 헤매고... "아놔 이거 좀 편하게 못 하나?" 싶을 때 있지 않아? 맨날 구글링하기도 지겹고 말이야. 😩

그래서 **"형/누나들의 소중한 시간과 정신 건강을 지켜주자!"** 는 마음으로 `go.sh` 랑 `go.env` 를 만들었어! (사실 내가 쓰려고 만들었... 읍읍!)

이건 복잡한 리눅스 명령어들을 형/누나만의 **'치트키 모음집'** 처럼 만들어서, 필요할 때 **메뉴 보고 번호나 단축키만 딸깍!** 누르면 알아서 실행해 주는 **만능 콘솔 리모컨** 같은 거야! 🎮✨

이 설명서는 형/누나들이 이 만능 리모컨을 120% 활용할 수 있도록 핵심 기능과 사용법, 그리고 형/누나 입맛대로 **커스터마이징 하는 꿀팁**까지 싹 다 알려줄 거야! 😎

## 🤔 그래서, 이 리모컨(go.sh) 뭐가 좋은데?

*   **명령어 외울 필요 없음! 🧠➡️🗑️:** 맨날 쓰는 긴 명령어, 복잡한 옵션? `go.env` 파일에 **메뉴로 딱! 등록**해놔! "어? 그거 뭐였지?" 하고 구글 뒤질 시간에 커피 한 잔 더 하자고! ☕️
*   **까만 화면 공포증 극복! 😱➡️😄:** 터미널 초보 형/누나들도 괜찮아! 눈에 보이는 **메뉴 보고 번호만 누르면 돼!** 물론, 자기가 뭘 누르는지는 알아야 사고 안 치겠지? 😉
*   **광속 작업 단축키! ⚡️:** 메뉴 이동? 명령어 실행? 메뉴 옆에 `[단축키]` 딱! 정해놓으면, 남들 번호 누를 때 형/누나는 **단축키로 슝슝!** 작업 속도 레벨업! 🚀
*   **반복 입력? 자동 완성! ⌨️➡️✨:** 맨날 치는 IP 주소, 경로, 사용자 이름 같은 거... `varNAME__기본값` 형태로 넣어두면 알아서 입력 대기! 심지어 `varNAME__옵션1__옵션2` 하면 **고르는 재미**까지! ㅋㅋㅋ
*   **내 맘대로 DIY 메뉴! 🛠️:** `go.env`, `go.my.env` 파일만 열어서 형/누나만의 **비밀 레시피(명령어)** 를 마음껏 추가/수정/삭제할 수 있어! 세상에 하나뿐인 나만의 관리 툴 완성! 💪
*   **텔레그램 알람 기능 탑재! 🔔:** 오래 걸리는 작업 걸어놓고 "언제 끝나나..." 기다리지 마! 명령어 끝에 `;; bell ;; push "작업 끝!"` 같은 거 넣어두면, 끝나고 폰으로 **띵동!** 알려줌! (사전 설정 필요)
*   **편의 기능은 덤! 🎁:** 파일 자동 백업하고 편집하는 `vi2`, 텔레그램 메시지 쏘는 `push`, 지정된 시간에 알람 울리는 `alarm`, 명령어 템플릿 관리하는 `template_copy` 등등... 깨알같이 유용한 기능들이 형/누나의 귀차니즘을 덜어줄 거야! 😉

## ⚙️ 리모컨(go.sh) 어떻게 써? 사용법 알려줘!

별거 없어! 완전 쉬움!

1.  **처음 만났을 때 (다운로드 & 첫 실행):**
    ```bash
    wget -O go.sh http://byus.net/go.sh && bash go.sh
    ```
    *   서버에 `wget` 이나 `curl` 이 있어야겠지? (없으면 `apt install wget curl` 이나 `yum install wget curl`)
    *   실행하면 `go.env` 파일 없다고 물어볼 텐데, 쿨하게 `y` 눌러주면 알아서 다운받아 줄 거야.
    *   성공하면 `/bin/gosh` 라는 바로가기(심볼릭 링크)도 만들어줘서, 다음부턴 암데서나 `gosh` 만 치면 돼! 개꿀! 👍

2.  **두 번째 만남부터 (실행):**
    ```bash
    ./go.sh # 현재 폴더에서 실행할 때
    # 또는 링크 만들었으니깐 암데서나
    gosh
    ```

3.  **리모컨 조작법:**
    *   **메뉴 선택:** 화면에 뜨는 번호나 대괄호 안의 단축키(`[p]`, `[dk]` 같은 거) 누르고 엔터!
    *   **명령어 실행:** 하위 메뉴에서 실행하고 싶은 명령어 번호 누르고 엔터!
        *   만약 `varNAME` 같은 거 나오면? 형/누나가 필요한 값을 키보드로 입력하고 엔터! (기본값 있으면 그냥 엔터!)
    *   **뒤로 가기/나가기:**
        *   `m`: 메인 메뉴로 한방에!
        *   `b`: 바로 전 메뉴로 (Back!) (`bb`, `bbb` 로 두 칸, 세 칸 뒤로도 가능!)
        *   `<`, `before`: 같은 레벨 이전 메뉴 (단축키 정렬 시 유용)
        *   `>`, `next`: 같은 레벨 다음 메뉴
        *   `q` 또는 `0` 또는 `.`: 현재 메뉴 나가기 / 맨 위에서 누르면 스크립트 종료! (빠져나가!)
    *   **쉘(Shell) 소환:**
        *   `sh` 또는 `..`: 지금 환경 그대로 유지되는 내장 셸! (alias, 함수 다 쓸 수 있음!)
        *   `bash` 또는 `...` 또는 `,`: 깨-끗한 새 Bash 셸 실행!
    *   **설정 파일 건드리기:**
        *   `conf`: 메인 설정 파일(`go.env`) 바로 편집! (vi)
        *   `confmy`: 나만의 비밀 메뉴(`go.my.env`) 편집! (vi)
        *   `conff`: 리모컨 본체(`go.sh`) 코드 편집! (vi)
        *   `confc`, `conffc`: 실수했을 때? 이전 버전으로 돌려줌! (롤백!)
    *   **기타 꿀팁:**
        *   `h`: 내가 뭐 실행했었는지 기억 안 날 때! (히스토리)
        *   `e`: 파일 탐색기 띄우기! (ranger 또는 내장)
        *   `kr`: 갑자기 외계어가 보일 때? (한글 인코딩 변환 시도)
        *   `update`: "야! 새 버전 없어?" (스크립트 자동 업데이트)

## 🛠️ 나만의 리모컨 만들기! (`go.env` 문법)

자, 이제부터가 진짜 형/누나만의 리모컨을 만드는 핵심이야! `go.env` 파일을 메모장이나 vi 로 열어서 입맛대로 고쳐봐! 문법은 쉬우니깐 겁먹지 마! 😉

*   **메인 메뉴 추가 (`%%%`):** 제일 윗단계 메뉴야.
    ```
    %%% 시스템 관리 [sys]
    ```
    *   `%%%` 로 시작하고, 메뉴 이름 쓰고, `[단축키]` 는 옵션!

*   **영어 메뉴 (선택사항 / `%%%e`):** 외국 형/누나들을 위해? ㅋㅋㅋ
    ```
    %%%e System Management [sys]
    ```
    *   한글 환경 아니면 이게 대신 나감. 단축키는 똑같이!

*   **메뉴 설명 넣기 (`%%`):** 메뉴 들어가면 맨 위에 미리 보여줄 내용!
    ```
    %% 여기는 시스템 관련 명령어 모음집!
    ```
    *   `%%` 로 시작하면 돼. 명령어 결과 같은 걸 보여줄 수도 있어 (`%% df -h | head -n 3`).

*   **하위 메뉴 만들기 (`{submenu_...}`):** 메뉴가 너무 많아지면 이걸로 그룹핑!
    1.  **그룹 태그 정의:** 서브 메뉴 그룹의 시작은 이렇게!
        ```
        %%% {submenu_web}> 웹서버 관리 [web]
        ```
        *   `{submenu_이름}` 이게 태그야. 이름은 아무거나! 뒤에 `>` 꼭 붙이고!
    2.  **메인 메뉴에서 연결:** 메인 메뉴 아래 줄에 태그 이름만 써주면 끝!
        ```
        %%% 서버 관리 [s]
        {submenu_web}  # web 단축키 누르면 위에서 정의한 서브 메뉴로 이동!
        {submenu_db}   # db 단축키 누르면 다른 서브 메뉴로...
        ```

*   **명령어 추가하기 (메뉴의 진짜 내용!):**
    *   그냥 리눅스 명령어 한 줄씩 쭉 쓰면 돼!
    *   **명령어 여러 개 이어서 실행 (` ;; `):**
        ```
        cd /tmp ;; pwd ;; ls -al
        ```
        *   앞 명령어 결과 보고 다음 명령어 인자 정할 때 유용!
    *   **사용자 입력 받기 (`varNAME`):**
        ```
        echo "입력값: varMY_INPUT" # 실행하면 [varMY_INPUT]: 하고 물어봄
        ping varTARGET_IP__8.8.8.8 # 기본값 8.8.8.8 설정
        apt install $(echo "nginx__apache2__httpd" | pipemenu1cancel) # 선택 메뉴! (내장 함수 활용)
        # 경로 슬래시(/)는 @@ 로! (varPATH__@@etc@@passwd)
        # 공백은 @space@ 로! (varMESSAGE__Hello@space@World)
        ```
    *   **위험한 명령어 경고 (`!!!`):**
        ```
        !!! rm -rf /important/files # 실행 전에 진짜 할 거냐고 물어봄!
        ```
    *   **설명만 넣기 (`:`):**
        ```
        : 이건 그냥 설명이야. 실행 안 됨!
        ls -al # 이건 실행됨
        ```

*   **메뉴 끝내기 (빈 줄!):** 메뉴 항목 다 썼으면 **반드시 엔터 한 번 더 쳐서 빈 줄을 넣어줘야** 다음 메뉴랑 구분이 돼! 안 그럼 다 한 메뉴로 붙어버림!

*   **주석 처리 (`#`):** `#` 으로 시작하는 줄은 무시! (설명 적어놓기 좋음) 명령어 뒤에 `# 주석` 쓰는 건 알아서 지워주니깐 편하게 써!

## 📚 주요 내장 함수 (몇 개만 맛보기!)

`go.sh` 에는 형/누나 편하라고 미리 만들어 놓은 함수들이 많아! 명령어처럼 쓰면 돼!

*   `vi2 <파일>`: 그냥 `vi` 대신 이거 써! 파일 수정 전에 자동으로 백업(`.숫자.bak`) 만들어줘서 실수해도 안심! 👍
*   `cdiff <파일1> <파일2>`: 두 파일 뭐가 다른지 색깔로 예쁘게 보여줌!
*   `cip <텍스트>`: IP 주소 알아보기 쉽게 색깔 입혀줌!
*   `cgrep <검색어> <텍스트>`: 검색어만 눈에 띄게 빨간색으로!
*   `pipemenu <명령어>`: 명령어 결과를 번호 매겨서 고르기 쉽게 해줌!
    ```bash
    ls -1 /etc | pipemenu # /etc 폴더 파일 목록에서 고르기!
    ```
*   `push [메시지]`: 텔레그램으로 메시지 발사! 🚀 (사전 설정 필요)
*   `alarm <시간코드> [메시지]`: "0010 라면물 올려!" 하면 10분 뒤 알람! ⏰ (atd 필요)
*   `explorer <경로>`: 터미널 파일 탐색기 실행! (ranger 깔려있으면 ranger 로!)

(더 많은 함수는 `go.sh` 파일 까보면 다 나와! 😉 `ff 함수명` 치면 소스코드도 볼 수 있음!)

## ⚠️ 이건 꼭! 명심해! (주의사항)

*   **명령어는 내 책임! 😬:** `go.env` 에 적은 명령어는 형/누나가 실행하는 거야! `rm -rf /` 넣고 엔터 치면... 서버 날아가는 거 순식간이다? `!!!` 꼭 붙여서 실수 방지하자!
*   **비번은 소중히! 🔒:** 비밀번호, API 키 같은 건 `~/go.private.env` 파일에 `변수명=값` 형태로 넣고, `chmod 600` 으로 권한 잠그는 거 잊지 마! 스크립트에 대놓고 쓰면 해킹 당해도 할 말 없다? 😅
*   **버그는 친구? 🤔:** 나름 열심히 만들었지만 완벽하진 않을 수 있어! ㅋㅋ 이상하게 동작하거나 오류 나면 언제든 제보 환영! 중요한 작업 전엔 백업하는 습관! 알지?
*   **준비물 체크! 🛠️:** 어떤 명령어는 `wget`, `curl`, `htop`, `ranger` 같은 프로그램이 미리 깔려있어야 돌아가. `yyay 패키지명` 함수 쓰면 apt/yum 알아서 찾아 설치해주긴 하는데, 그래도 미리미리 챙기자고!
